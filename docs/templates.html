Como os templates funcionamHow templates work

Princípio de composiçãoComposition principle
Templates são provas de conceito: mostram que qualquer interface de negócio pode ser construída apenas com componentes DS existentes, classes utilitárias e tokens de espaçamento. Nenhum estilo visual novo é adicionado — apenas estrutura de layout usa CSS próprio, sempre referenciando tokens var(--ds-*). Templates are proof-of-concept: they show that any business interface can be built using only existing DS components, utility classes, and spacing tokens. No new visual styles are added — only layout structure uses its own CSS, always referencing var(--ds-*) tokens.
Sincronização automáticaAutomatic sync
Os templates importam apenas ../css/design-system.css. Qualquer mudança nos tokens ou componentes do DS é refletida automaticamente em todos os templates sem nenhuma edição. Templates import only ../css/design-system.css. Any change to DS tokens or components is automatically reflected across all templates without any edits.

Estrutura de arquivosFile structure

Cada template tem dois artefatos: uma página de documentação (docs/template-*.html) com contexto de uso e uma página standalone (templates/*.html) renderizando o template em tela cheia. Each template has two artifacts: a documentation page (docs/template-*.html) with usage context and a standalone page (templates/*.html) rendering the full-screen template.
