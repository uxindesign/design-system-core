Control SizingControl Sizing

Button, Input, Select e Textarea compartilham um sistema dimensional unificado. Mesma altura, mesmo padding, mesmo tamanho de ícone em cada nível -- para que os controles se alinhem perfeitamente lado a lado. Definido no ADR-006.Button, Input, Select, and Textarea share a unified dimensional system. Same height, same padding, same icon size at each tier -- so controls align perfectly when placed side by side. Defined in ADR-006.

A fórmulaThe formula

height = padding-y × 2 + line-height

Todo controle interativo segue esta fórmula. O line-height é igual ao tamanho do ícone em cada nível, garantindo que texto e ícones se alinhem verticalmente.Every interactive control follows this formula. The line-height equals the icon size at each tier, ensuring text and icons align vertically.

Cores toned (ADR-007)Toned colors (ADR-007)

A variante "Toned" do Button usa overlays coloridos translúcidos. Isso é diferente de subtle (opaco, step 100 da paleta) e muted (opaco, step 50 da paleta):The Button "Toned" variant uses translucent colored overlays. This is different from subtle (opaque, palette step 100) and muted (opaque, palette step 50):

A camada Foundation tem tokens de overlay colorido (--ds-overlay-blue-600-* para light, --ds-overlay-blue-400-* para dark). Tokens semânticos fazem alias destes por modo. O padrão é extensível para cores de feedback quando necessário.Foundation layer has colored overlay tokens (--ds-overlay-blue-600-* for light, --ds-overlay-blue-400-* for dark). Semantic tokens alias these per mode. The pattern is extensible to feedback colors when needed.
