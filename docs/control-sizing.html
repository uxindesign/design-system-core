Control Sizing

Button, Input, Select, and Textarea share a unified dimensional system. Same height, same padding, same icon size at each tier — so controls align perfectly when placed side by side. Defined in ADR-006.

The formula

height = padding-y × 2 + line-height

Every interactive control follows this formula. The line-height equals the icon size at each tier, ensuring text and icons align vertically.

Token layers

Control tokens live in the semantic layer. Component tokens reference them:

/* Semantic (shared) */ --ds-size-control-md: 2.5rem; /* 40px */ --ds-space-control-padding-x-md: var(--ds-spacing-4); /* 16px */ --ds-space-control-padding-y-md: var(--ds-spacing-2-5); /* 10px */ /* Component (per-component alias) */ --ds-button-height-md: var(--ds-size-control-md); --ds-button-padding-x-md: var(--ds-space-control-padding-x-md); --ds-input-height-md: var(--ds-size-control-md); --ds-input-padding-x-md: var(--ds-space-control-padding-x-md);

Toned colors (ADR-007)

The Button "Toned" variant uses translucent colored overlays. This is different from subtle (opaque, palette step 100) and muted (opaque, palette step 50):

Token patternVisualUse case
color.primary.subtleOpaque light blue (blue-100)Subtle backgrounds, badges
color.primary.mutedOpaque lighter blue (blue-50)Muted backgrounds, selected states
color.primary.toned.*Translucent blue (12-28% opacity)Button Toned variant — surface shows through

Foundation layer has colored overlay tokens (--ds-overlay-blue-600-* for light, --ds-overlay-blue-400-* for dark). Semantic tokens alias these per mode. The pattern is extensible to feedback colors when needed.
