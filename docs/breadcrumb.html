Breadcrumb

Navigation aid showing the user's current location within a page hierarchy. Breadcrumbs help users understand where they are and navigate back to parent pages.

When to use

Use breadcrumbs when
The site has hierarchical navigation and the user needs to see their current location and navigate back to parent pages. Ideal for content-heavy sites with 3+ levels of depth.
Don't use breadcrumbs when
The site is flat (1-2 levels only), the flow is linear (use a back button instead), or there is only a single level above the current page.

Anatomy

1 Home 4 / 6 Products / Sneakers 5 2 3
1 Nav — <nav> element with aria-label="Breadcrumb".
2 List (.ds-breadcrumb) — <ol> containing breadcrumb items.
3 Item (.ds-breadcrumb__item) — individual breadcrumb entry.
4 Link — navigable parent pages.
5 Current (.ds-breadcrumb__current) — text (not a link), with aria-current="page".
6 Separator — CSS-generated or decorative text between items.

Basic breadcrumb

A simple 3-level hierarchy. The last item is the current page and is not a link.

Home / Products / Sneakers 
<nav class="ds-breadcrumb" aria-label="Breadcrumb">
  <a href="/home" class="ds-breadcrumb__item">Home</a>
  <span class="ds-breadcrumb__separator">/</span>
  <a href="/products" class="ds-breadcrumb__item">Products</a>
  <span class="ds-breadcrumb__separator">/</span>
  <span class="ds-breadcrumb__item ds-breadcrumb__item--current"
        aria-current="page">Sneakers</span>
</nav>

Deep hierarchy

A 5-level breadcrumb trail for deeply nested pages.

Home / Dashboard / Settings / Account / Security 
<nav class="ds-breadcrumb" aria-label="Breadcrumb">
  <a href="#" class="ds-breadcrumb__item">Home</a>
  <span class="ds-breadcrumb__separator">/</span>
  <a href="#" class="ds-breadcrumb__item">Dashboard</a>
  <span class="ds-breadcrumb__separator">/</span>
  <a href="#" class="ds-breadcrumb__item">Settings</a>
  <span class="ds-breadcrumb__separator">/</span>
  <a href="#" class="ds-breadcrumb__item">Account</a>
  <span class="ds-breadcrumb__separator">/</span>
  <span class="ds-breadcrumb__item ds-breadcrumb__item--current"
        aria-current="page">Security</span>
</nav>

Two levels

The minimum useful breadcrumb with just a parent and the current page.

Blog / How to get started 
<nav class="ds-breadcrumb" aria-label="Breadcrumb">
  <a href="/blog" class="ds-breadcrumb__item">Blog</a>
  <span class="ds-breadcrumb__separator">/</span>
  <span class="ds-breadcrumb__item ds-breadcrumb__item--current"
        aria-current="page">How to get started</span>
</nav>

Best practices

Home / Products / Sneakers
Do
Show the full hierarchy path from root. Current page as text (not a link).
Home / ... / Sneakers
Don't
Truncate intermediate levels. Don't make the current page clickable.

Content guidelines

RuleExample
Labels match page titles exactly"Products" in breadcrumb matches the "Products" page title
Use sentence case"Getting started" — not "Getting Started" or "GETTING STARTED"
Keep hierarchy to 3-5 levelsHome / Dashboard / Settings / Security — not 8+ levels deep

CSS classes

ds-breadcrumb__separatorVisual separator between items (e.g. "/")

Keyboard interaction

KeyAction
TabMoves focus between breadcrumb links (current page is not focusable)
EnterNavigates to the focused link

The current page item is a <span>, not a link, so it is naturally excluded from the tab order. Separators are decorative and not focusable.

Accessibility

WCAG criterionRequirementStatus
1.3.1 Info and Relationships (A)<nav> landmark with aria-label="Breadcrumb". Uses <ol> for semantic list structure.✓
2.4.8 Location (AAA)Communicates user's current location within the site hierarchy.✓
4.1.2 Name, Role, Value (A)aria-current="page" on the current page item.✓
ARIA attributes summary
aria-label="Breadcrumb" — on the <nav> element to identify the landmark.
aria-current="page" — on the current page item to communicate location to screen readers.
Separators should be purely decorative (CSS-generated or aria-hidden="true").
