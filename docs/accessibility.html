Accessibility

WCAG 2.2 AA compliance across all components.

Focus Management

All interactive components use a double-ring focus indicator via the --ds-focus-ring token. This approach ensures the focus ring is visible against any background color, meeting WCAG 2.2 Success Criterion 2.4.7 (Focus Visible).

Tab through these elements to see the focus ring:

/* Focus ring token — double ring for contrast */
--ds-focus-ring: 0 0 0 2px var(--ds-surface-default),
                 0 0 0 4px var(--ds-color-primary);

/* Applied via :focus-visible on all interactive elements */
.ds-btn:focus-visible,
.ds-input:focus-visible {
  outline: none;
  box-shadow: var(--ds-focus-ring);
}
Why :focus-visible?

We use :focus-visible instead of :focus so the ring only appears during keyboard navigation, not on mouse clicks. This gives keyboard users clear indicators without adding visual noise for pointer users.

Color Contrast

All color pairings in the design system meet or exceed WCAG 2.2 AA contrast requirements in both light and dark modes.

Element Type Minimum Ratio WCAG Criterion
Normal text (< 18px) 4.5 : 1 1.4.3 Contrast (Minimum)
Large text (≥ 18px or 14px bold) 3 : 1 1.4.3 Contrast (Minimum)
UI components (borders, icons) 3 : 1 1.4.11 Non-text Contrast
Focus indicators 3 : 1 2.4.11 Focus Appearance

Keyboard Navigation

Every interactive component is fully operable via keyboard. The following keys are supported:

Key Action
Tab Move focus to the next interactive element
Shift + Tab Move focus to the previous interactive element
Enter / Space Activate buttons, toggles, checkboxes, and radio buttons
Arrow keys Navigate within tabs, radio groups, and select dropdowns
Escape Close modals, tooltips, and dropdowns; return focus to trigger

Reduced Motion

Users who prefer reduced motion (via OS settings) will see all transitions and animations removed or shortened. This respects WCAG 2.3.3 (Animation from Interactions). 

@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}

Touch Targets

All interactive elements maintain a minimum touch target size of 44 × 44 CSS pixels, in line with WCAG 2.5.8 (Target Size). This applies to buttons, checkboxes, radio buttons, toggles, and other clickable controls.

  • Small buttons (ds-btn--sm) use padding to meet the 44px minimum height.
  • Checkbox and radio inputs have a clickable label area that satisfies the target size.
  • Icon-only buttons are sized at minimum 44px via internal padding.

ARIA Attributes

Components that require ARIA attributes for screen-reader compatibility:

Component Required ARIA
Button (icon-only) aria-label
Button (loading) aria-disabled="true", aria-busy="true"
Toggle role="switch", aria-checked
Modal role="dialog", aria-modal="true", aria-labelledby
Tooltip role="tooltip", aria-describedby
Tabs role="tablist", role="tab", role="tabpanel", aria-selected, aria-controls
Alert role="alert" or role="status"
Breadcrumb aria-label="Breadcrumb", aria-current="page" on current item
Spinner role="status", aria-label="Loading"
Form Field (error) aria-invalid="true", aria-describedby pointing to error message
Avatar (decorative) aria-hidden="true" or alt="" for images
