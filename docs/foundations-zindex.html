Z-index

6 z-index tokens that define the stacking order of positioned elements. The scale uses increments of 10 to leave room for intermediate values when needed.

Stacking context

Visual representation of how layers stack. Higher z-index = closer to the user.

z-0 · page content
z-10 · sticky headers
z-20 · dropdowns, popovers
z-30 · sidesheets, overlays
z-40 · modals, dialogs
z-50 · toast, notifications

Tokens

TokenValueLayerComponents
--ds-z-00Base contentDefault page flow, cards, sections
--ds-z-1010StickySticky headers, table headers, fixed toolbars
--ds-z-2020DropdownDropdowns, popovers, tooltips, select menus
--ds-z-3030OverlaySidesheets, navigation overlays, backdrop
--ds-z-4040ModalModals, dialogs, confirmation prompts
--ds-z-5050TopmostToast notifications, global loading indicators

Relationship with elevation

Z-index controls stacking order. Elevation (shadows + surfaces) controls visual depth. They often correlate but are independent systems:

LayerZ-indexShadowSurface
Cards, panels--ds-z-0--ds-shadow-sm--ds-surface-raised
Dropdowns--ds-z-20--ds-shadow-md--ds-surface-overlay
Modals--ds-z-40--ds-shadow-xl--ds-surface-elevated

Rules

Always use tokens, never magic numbers
Hardcoded z-index values (z-index: 9999) break the stacking contract and cause bugs that are hard to trace. If a component needs to stack above another, it should use the next token in the scale. If the existing scale doesn't cover a case, that's a signal to evaluate whether a new token is needed (which requires an ADR).
RuleRationale
Only use --ds-z-* tokensPredictable stacking across the entire app
Tooltips and dropdowns share --ds-z-20They never coexist — opening one closes the other
Modal backdrop uses --ds-z-30, modal content uses --ds-z-40Backdrop sits below the dialog but above all other content
Toasts always on top (--ds-z-50)Notifications must be visible even over modals
Increments of 10Leaves room for intermediate values if edge cases arise
