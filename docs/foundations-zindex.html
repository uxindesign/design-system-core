Z-index

6 tokens de z-index que definem a ordem de empilhamento de elementos posicionados. A escala usa incrementos de 10 para deixar espaco para valores intermediarios quando necessario.6 z-index tokens that define the stacking order of positioned elements. The scale uses increments of 10 to leave room for intermediate values when needed.

Contexto de empilhamentoStacking context

Representação visual de como as camadas se empilham. Maior z-index = mais proximo do usuário.Visual representation of how layers stack. Higher z-index = closer to the user.

Relação com elevaçãoRelationship with elevation

Z-index controla a ordem de empilhamento. Elevação (sombras + superfícies) controla a profundidade visual. RegrasRules

Sempre use tokens, nunca numeros magicosAlways use tokens, never magic numbers
Valores de z-index hardcoded (z-index: 9999) quebram o contrato de empilhamento e causam bugs difíceis de rastrear. Se um componente precisa ficar acima de outro, deve usar o proximo token na escala. Se a escala existente não cobre um caso, isso e um sinal para avaliar se um novo token é necessario (o que requer uma ADR).Hardcoded z-index values (z-index: 9999) break the stacking contract and cause bugs that are hard to trace. If a component needs to stack above another, it should use the next token in the scale. If the existing scale doesn't cover a case, that's a signal to evaluate whether a new token is needed (which requires an ADR).
