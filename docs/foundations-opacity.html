Opacity

7 opacity tokens for controlling transparency. Used in disabled states, overlays, and layered surfaces.

Scale

Each swatch shows the primary color at the given opacity over a checker pattern.

--ds-opacity-00 — fully transparent
--ds-opacity-50.05 — barely visible, subtle hover overlays
--ds-opacity-100.10 — pressed state overlays
--ds-opacity-250.25 — skeleton loading, placeholder content
--ds-opacity-500.50 — disabled foreground (text/icons)
--ds-opacity-750.75 — overlay backgrounds (modals, drawers)
--ds-opacity-1001 — fully opaque (default)

Usage in the DS

ContextTokenWhy
Disabled buttons, inputs--ds-opacity-50Dims the entire element while preserving its layout and size
Modal/dialog backdrop--ds-opacity-75Darkens the page behind the modal without fully hiding it
Skeleton loading--ds-opacity-25Creates subtle placeholder shapes that indicate loading
Ghost button hover--ds-opacity-5Tints the background just enough to signal interactivity

Relationship with overlay tokens

Opacity tokens control the opacity CSS property on an element. Overlay tokens (--ds-color-overlay-black-*, --ds-color-overlay-white-*) use rgba() values and are applied as background-color. They serve different purposes:

ApproachToken exampleAffectsUse when
Opacity--ds-opacity-50Entire element + childrenDisabling a complex element (button with icon + text)
Overlay--ds-color-overlay-black-5Background onlyHover/pressed tint on interactive surfaces
