Motion

Duration and easing tokens that control transitions and animations across all components. 7 tokens total: 3 durations + 4 easings.

Duration

TokenValueUsage
--ds-duration-fast100msMicro-interactions: color changes on hover, opacity shifts, icon swaps
--ds-duration-normal200msStandard transitions: button states, input focus, border color changes
--ds-duration-slow300msLarger movements: modals, sidesheets, accordion expand/collapse

Usage in components

Combine a duration token with an easing token on the CSS transition shorthand:

/* Button hover transition */ .ds-btn { transition: background-color var(--ds-duration-fast) var(--ds-ease-default), color var(--ds-duration-fast) var(--ds-ease-default), border-color var(--ds-duration-fast) var(--ds-ease-default); } /* Modal enter */ .ds-modal__content { transition: opacity var(--ds-duration-slow) var(--ds-ease-out), transform var(--ds-duration-slow) var(--ds-ease-out); }

Reduced motion

prefers-reduced-motion
Users who enable reduced motion in their OS settings will have all transitions and animations shortened to near-zero. This is enforced globally in the base reset — individual components do not need to handle it.
@media (prefers-reduced-motion: reduce) { *, *::before, *::after { animation-duration: 0.01ms !important; animation-iteration-count: 1 !important; transition-duration: 0.01ms !important; scroll-behavior: auto !important; } }

WCAG reference: 2.3.3 Animation from Interactions (AAA). While the DS targets AA, this is implemented because the cost is near-zero and the impact for motion-sensitive users is significant.
