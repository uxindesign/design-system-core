Toggle

Toggles (switches) let users turn a setting on or off instantly. Built with a styled native checkbox and role="switch".

When to use

Use toggles when
The user needs to turn an on/off setting that takes effect immediately. Dark mode, notifications, feature flags, auto-save preferences.
Don't use toggles when
The change requires form submission — use a Checkbox instead. When there are more than two options — use Radio or Select.

Anatomy

1 2 3 4
1 Input (.ds-toggle) — styled <input type="checkbox"> with role="switch".
2 Track — CSS background that changes color based on checked state.
3 Thumb — CSS pseudo-element that slides left/right.
4 Label (.ds-toggle-label) — wraps toggle + descriptive text.

Best practices

Do
Labels describe the setting: "Dark mode", "Email notifications".
Don't
"Enable dark mode" is redundant with the toggle state itself.
Do
Place toggle right of label. Effect is immediate.
Don't
Don't use toggle for actions needing confirmation. Use a Button + dialog instead.

Content guidelines

RuleExample
Use a noun phrase for the label"Dark mode", "Auto-save", "Email notifications"
Don't use "Enable/Disable""Notifications" — not "Enable notifications"
Keep labels short (1-3 words)"Auto-save" — not "Automatically save my work every 5 minutes"

Keyboard interaction

KeyAction
TabMoves focus to the toggle
SpaceToggles on/off
EnterDoes NOT toggle (standard switch behavior)

Disabled toggles ([disabled]) are removed from tab order automatically by the browser.

ARIA attributes summary
role="switch" — required on the checkbox so screen readers announce it as a switch.
aria-checked — managed automatically by the native <input type="checkbox">.
A visible label should always describe what the toggle controls.
