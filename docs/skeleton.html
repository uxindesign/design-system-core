Skeleton

Placeholder shapes with a pulse animation, used to indicate loading content. Skeletons provide a low-fidelity preview of the layout before data arrives.

When to use

Use skeletons when
Content is loading and the layout structure is known ahead of time: pages, cards, lists, tables, and profile sections. Skeletons preview the shape of incoming content so the interface feels faster.
Don't use skeletons when
A single action is loading (use a Spinner in the button instead) or when the content structure is unknown and unpredictable (use a Spinner).

Text skeleton

Represents lines of text content. Use varying widths to simulate natural paragraph shapes.

<div aria-busy="true" aria-label="Loading text content">
  <div class="ds-skeleton ds-skeleton--text" style="width:100%;height:14px;"></div>
  <div class="ds-skeleton ds-skeleton--text" style="width:85%;height:14px;"></div>
  <div class="ds-skeleton ds-skeleton--text" style="width:70%;height:14px;"></div>
</div>

Circle skeleton

Perfect for avatar or icon placeholders.

<div class="ds-skeleton ds-skeleton--circle" style="width:32px;height:32px;"></div>
<div class="ds-skeleton ds-skeleton--circle" style="width:40px;height:40px;"></div>
<div class="ds-skeleton ds-skeleton--circle" style="width:56px;height:56px;"></div>

Rectangle skeleton

Full-width block for images, banners, or large content areas.

<div class="ds-skeleton ds-skeleton--rectangle" style="width:100%;height:120px;"></div>

Composition: profile card

Combining different skeleton shapes to create a realistic loading placeholder for a profile card.

<div aria-busy="true" aria-label="Loading profile card">
  <!-- Avatar + name -->
  <div style="display:flex;align-items:center;gap:var(--ds-spacing-3);">
    <div class="ds-skeleton ds-skeleton--circle" style="width:48px;height:48px;"></div>
    <div>
      <div class="ds-skeleton ds-skeleton--text" style="width:60%;height:14px;"></div>
      <div class="ds-skeleton ds-skeleton--text" style="width:40%;height:12px;"></div>
    </div>
  </div>
  <!-- Bio -->
  <div class="ds-skeleton ds-skeleton--text" style="width:100%;height:12px;"></div>
  <div class="ds-skeleton ds-skeleton--text" style="width:90%;height:12px;"></div>
  <!-- Actions -->
  <div class="ds-skeleton ds-skeleton--rectangle" style="width:100px;height:36px;"></div>
</div>

Composition: card list

A loading state for a list of content cards with image, title, and description.

<div aria-busy="true" aria-label="Loading card list">
  <div class="card">
    <div class="ds-skeleton ds-skeleton--rectangle" style="height:140px;"></div>
    <div class="ds-skeleton ds-skeleton--text" style="width:60%;height:16px;"></div>
    <div class="ds-skeleton ds-skeleton--text" style="width:100%;height:12px;"></div>
    <div class="ds-skeleton ds-skeleton--text" style="width:90%;height:12px;"></div>
    <div style="display:flex;align-items:center;gap:var(--ds-spacing-2);">
      <div class="ds-skeleton ds-skeleton--circle" style="width:24px;height:24px;"></div>
      <div class="ds-skeleton ds-skeleton--text" style="width:80px;height:12px;"></div>
    </div>
  </div>
</div>

Best practices

Do
Match skeleton shapes to actual content: circles for avatars, text lines for paragraphs, rectangles for images.
Don't
Don't use a single generic rectangle for all loading states -- it doesn't preview the real layout.

Accessibility

WCAG criterionRequirementStatus
4.1.2 Name, Role, Value (A)aria-busy="true" on the loading container + aria-label describing what is loading✓
1.3.1 Info and Relationships (A)aria-hidden="true" on individual skeleton elements (they are decorative)✓
2.3.3 Animation from Interactions (AAA)Respects prefers-reduced-motion -- pulse animation paused or removed✓
ARIA attributes summary
aria-busy="true" — set on the container that holds skeleton placeholders while loading. Remove it once real content appears.
aria-hidden="true" — set on individual skeleton shapes (they are visual placeholders, not meaningful content).
prefers-reduced-motion — the skeleton CSS respects this media query by pausing the pulse animation.
