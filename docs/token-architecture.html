Token ArchitectureToken Architecture

Design tokens são organizados em três camadas. Cada camada tem um papel específico, e tokens em uma camada referenciam a camada abaixo -- nunca o contrário.Design tokens are organized in three layers. Each layer has a specific role, and tokens in one layer reference the layer below -- never the other way around.

Três camadasThree layers

Foundation
Valores brutos. A única camada com valores absolutos (hex, rem, px).Raw values. The only layer with absolute values (hex, rem, px).

foundation.color.blue.600 → #2563EB
foundation.spacing.4 → 1rem
foundation.radius.md → 8px

~217 tokens em 10 categorias.~217 tokens across 10 categories.
→
Semantic
Intenção e significado. Referencia foundation via aliases. Sensível a modo (light/dark).Intent and meaning. References foundation via aliases. Mode-aware (light/dark).

semantic.color.primary.default → {foundation.brand.primary}
semantic.text.default → {foundation.color.neutral.900} (light)
semantic.text.default → {foundation.color.neutral.50} (dark)

~104 tokens × 2 modos.~104 tokens × 2 modes.
→
Component
Tokens por componente. Referencia semantic, nunca foundation.Per-component tokens. References semantic, never foundation.

component.button.background.brand.default → {semantic.color.primary.default}
component.button.padding.md → {semantic.space.inset.md}

118 tokens em 11 arquivos de componente.118 tokens across 11 component files.

Como funciona a cadeia de aliasHow an alias chain works

Quando você clica em um button Brand, sua cor de fundo é resolvida por esta cadeia:When you press a Brand button, its background color resolves through this chain:

component button.background.brand.default
↓ references
semantic semantic.color.primary.default
↓ references
brand foundation.brand.primary
↓ references
foundation foundation.color.blue.600 → #2563EB

Para mudar a marca, basta trocar o brand.json. Toda a cadeia se atualiza: cada componente que usa primary.default recebe a nova cor sem tocar em nenhum outro arquivo.To rebrand, you swap brand.json. The entire chain updates: every component that uses primary.default gets the new color without touching any other file.

Brand SublayerBrand sublayer

Brand fica dentro de Foundation como uma ponte entre paleta e intenção. Tem apenas 2 tokens -- sem estados, sem modos.Brand sits inside Foundation as a bridge between palette and intent. It has only 2 tokens -- no states, no modes.

{ "foundation": { "brand": { "primary": { "$value": "{foundation.color.blue.600}" }, "secondary": { "$value": "{foundation.color.purple.600}" } } } }

Apenas semantic.color.primary.default referencia foundation.brand.primary. Todos os outros estados (hover, active, subtle) referenciam a paleta foundation diretamente, porque mudanças de estado precisam de steps específicos da paleta que não seguem uma fórmula.Only semantic.color.primary.default references foundation.brand.primary. All other states (hover, active, subtle) reference the foundation palette directly, because state shifts need specific palette steps that don't follow a formula.

RegrasRules

#RegraRule
1Tokens de componente referenciam semantic, nunca foundation. Exceção: estados hover/active que precisam de um step específico da paleta.Component tokens reference semantic, never foundation. Exception: hover/active states that need a specific palette step.
2Tokens semânticos referenciam foundation, nunca hardcoded. Nenhum hex ou rgba bruto na camada semântica.Semantic tokens reference foundation, never hardcoded. No raw hex or rgba in the semantic layer.
3Foundation é a única camada com valores absolutos.Foundation is the only layer with absolute values.
4Brand faz parte da foundation: 2 tokens (primary, secondary), sem estados, trocável por tema.Brand is part of foundation: 2 tokens (primary, secondary), no states, swappable per theme.
5Todo token tem $type conforme spec DTCG.Every token has $type per DTCG spec.
6Tokens não óbvios requerem $description.Non-obvious tokens require $description.
7light.json e dark.json devem ter exatamente o mesmo conjunto de chaves.light.json and dark.json must have exactly the same key set.
8Novas categorias ou quebras de hierarquia requerem um ADR.New categories or hierarchy breaks require an ADR.
