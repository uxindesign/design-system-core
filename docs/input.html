Input

Text inputs allow users to enter and edit text. Uses a wrapper pattern with an inner field element for consistent styling.

When to use

Use when
The user needs to enter a short, single-line text value: name, email, phone, URL, search query, password.
Don't use when
The expected input is multi-line (use Textarea), the user must pick from a fixed set of options (use Select, Checkbox, or Radio), or the value is binary on/off (use Toggle).
SituationUse instead
Multi-line textTextarea
Choosing from a listSelect
Yes/no or on/offToggle or Checkbox
Searching with autocompleteCustom search component (future)

Anatomy

search Search...
1 2 3
1 Wrapper (.ds-input) — container with border, radius, background.
2 Icon (optional, .ds-input__icon) — leading icon, 16px, decorative.
3 Field (.ds-input__field) — the native <input> element.
4 Label — external, via Form Field wrapper (.ds-field__label).
5 Helper text — external, via Form Field (.ds-field__helper).
6 Error message — external, via Form Field (.ds-field__error).

Best practices

Do
Always use inside a Form Field wrapper with a visible label.
Don't
Use a placeholder as the only label — it disappears when the user starts typing.
Do
Match the input size to the expected content length. Use full-width for emails, shorter widths for zip codes.
Don't
Make all inputs the same width regardless of content.
Email is required. Enter a valid email address.
Do
Show error state with ds-input--error AND ds-field--error, with aria-invalid="true" and a clear error message.
Don't
Show error styling without an error message — the user needs to know what went wrong.

Content guidelines

RuleExample
Placeholder text should be an example value, not an instruction"john@example.com" — not "Enter your email"
Labels should be 1-3 words, noun or noun phrase"Email address", "Phone number"
Helper text is optional, use for format hints"Must be at least 8 characters"
Error messages should say what went wrong AND how to fix it"Email is required. Enter a valid email address."
Required fields: add required indicator to the label<span class="ds-field__required">*</span>

ARIA attributes summary
for / id — always pair <label> with <input> using matching attributes.
aria-invalid="true" — set on the native input when in error state.
aria-describedby — link the input to the error message element ID so screen readers announce the error.
autocomplete — use appropriate values (name, email, tel, etc.) for personal data fields.
