Form Field

Structured form field wrapper that provides labels, helper text, required indicators, and error messages for form inputs. Use it to create consistent, accessible form layouts.

When to use

Use Form Field when
Wrapping any form input that needs a label, helper text, required indicator, or error message. Provides consistent spacing and accessibility associations.
Don't use Form Field when
The input is standalone without a label (rare). For Checkbox or Radio groups, use a <fieldset> with <legend> instead.

Anatomy

We'll never share your email. Email is required.
1 2 3 4 5 6
1 Wrapper (.ds-field) — vertical stack container.
2 Label (.ds-field__label) — associated via for/id.
3 Required (.ds-field__required) — asterisk indicator.
4 Input slot — accepts Input, Textarea, or Select.
5 Helper (.ds-field__helper) — format hints or instructions.
6 Error (.ds-field__error) — validation error message.

Basic field

We'll never share your email with anyone. 
<div class="ds-field">
  <label class="ds-field__label" for="field-email">Email address</label>
  <input class="ds-input__field" type="email" id="field-email"
         placeholder="you@example.com"
         aria-describedby="field-email-helper">
  <span class="ds-field__helper" id="field-email-helper">
    We'll never share your email with anyone.
  </span>
</div>

Best practices

Do
Always use Form Field for label + input combinations.
Email
Don't
Don't build custom wrappers. Use the standardized Form Field pattern.
Must contain at least 8 characters. Password is too short.
Do
Show both error and helper text in error state.
Password is too short.
Don't
Don't remove the helper when an error appears. Users lose context.

Keyboard interaction

Form Field is a layout container only. It has no keyboard interactions of its own. Keyboard behavior is inherited from the child input component (Input, Textarea, or Select).

Accessibility

WCAG criterionRequirementStatus
1.3.1 Info and Relationships (A)Label via for/id, error via aria-describedby✓
3.3.1 Error Identification (A)ds-field--error + ds-field__error + aria-invalid✓
3.3.2 Labels or Instructions (A)Visible label always present✓
3.3.3 Error Suggestion (AA)Error messages include correction hint✓
ARIA attributes summary
for/id — associates label with input.
aria-describedby — links helper text and error messages to the input.
aria-required="true" — set on required inputs.
aria-invalid="true" — set on inputs in an error state.
